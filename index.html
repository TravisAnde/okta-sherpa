Okta Workforce Identity User Guide

Environment Setup

This guide uses the following machines:

  • CL03: General client machine
  • CL04: General client machine
  • DC01: Primary Domain Controller (AD Agent)
  • DC02: Secondary Domain Controller

Section 1: Creating Demo Organization

  1. Log into CL03 with your provided credentials
  2. Navigate to the Okta Partner Portal (https://partners.okta.com)
  3. Click on "Create Demo Org" in the dashboard
  4. Fill in the required information:
    • Organization name
    • Primary administrator email
    • Region selection
    • Use case selection
  5. Submit the form and wait for confirmation email
  6. Follow the activation link in the email to set up your administrator password
Note: Keep your administrator credentials secure. You'll need them for the following sections.

Section 2: Active Directory Integration

  1. Log into your Okta admin console
  2. Navigate to Directory > Directory Integrations
  3. Click "Add Directory" and select "Active Directory"
  4. Download the Okta AD agent installer
  5. On DC01, run the AD agent installer with administrator privileges
  6. During installation, provide:
    • Your Okta tenant URL
    • Admin credentials
    • AD service account details
  7. Verify the connection in Okta admin console
  8. Configure AD synchronization settings
Note: Ensure DC01 has outbound connectivity to your Okta tenant.

Section 3: User and Group Management

  1. Creating Okta-Managed Users:
    • Navigate to Directory > People
    • Click "Add Person"
    • Create user 1 with details: First Name: Test1 Last Name: User1 Username: test1@yourdomain.com
    • Repeat for user 2
  2. Creating Okta-Managed Groups:
    • Navigate to Directory > Groups
    • Click "Add Group"
    • Create "IT-Admin" group
    • Create "General-Users" group
  3. Assign users to groups

Section 4: Application Integration

  1. Navigate to Applications > Applications
  2. Click "Browse App Catalog"
  3. Add the following applications:
    • Microsoft 365
    • Salesforce
    • Slack
    • Zoom
    • AWS Single Sign-On
  4. For each application:
    • Configure SSO settings
    • Set up provisioning (if applicable)
    • Assign to appropriate groups
Note: Each application may require specific configuration details from the service provider.

Section 5: Authentication Policies

  1. Navigate to Security > Authentication
  2. Configure Global Session Policy:
    • Set session lifetime
    • Configure sign-on requirements
  3. Create Authentication Policies:
    • High-security policy for IT-Admin group
    • Standard policy for General-Users group
  4. Configure MFA Settings:
    • Enable Okta Verify
    • Set up factor enrollment requirements
    • Configure factor enrollment policy
  5. Test authentication flows
