Authentication Policies

⏱️ 30-40 minutes

Section Prerequisites

  • Completed application integrations
  • User groups configured
  • MFA factors determined
  • Security requirements documented

Part 1: Global Session Policy

  1. Configure Session Settings

    Navigate to Security > Global Session Policy

    • Set session lifetime: Max session lifetime: 8 hours Idle timeout: 30 minutes
    • Configure session persistency settings
    • Set refresh token lifetime
    Balance security with user experience when setting timeouts.
  2. Configure Sign-On Policy

    Under Security > Authentication Policies:

    • Set password complexity requirements
    • Configure lockout settings
    • Set password expiry rules

Part 2: MFA Configuration

  1. Enable MFA Factors

    Navigate to Security > Multifactor:

    Factor Status User Groups
    Okta Verify Required All Users
    SMS Authentication Optional General-Users
    Security Question Required All Users
  2. Configure Enrollment Rules
    1. Set enrollment requirements:
      • Required factors
      • Optional factors
      • Enrollment timeframe
    2. Configure enrollment reminders
    3. Set grace period settings

Part 3: Authentication Policies

  1. Create High-Security Policy

    For IT-Admin group:

    Policy Name: High Security Access Applies to: IT-Admin group Requirements: - Password + Okta Verify - No remembered devices - Session timeout: 4 hours
  2. Create Standard Policy

    For General-Users group:

    Policy Name: Standard Access Applies to: General-Users group Requirements: - Password + Any MFA - Allow remembered devices - Session timeout: 8 hours
  3. Configure Risk-Based Authentication

    Set up adaptive policies:

    • Location-based access rules
    • Device context policies
    • Network zone configurations
    • Suspicious activity detection

Part 4: Password Policies

  1. Configure Password Requirements
    Requirement Setting
    Minimum length 12 characters
    Character requirements 1 upper, 1 lower, 1 number, 1 symbol
    Password age 90 days
    Password history 24 previous passwords

Verification Checklist

  • Global session policy configured
  • MFA factors enabled and tested
  • Authentication policies applied to correct groups
  • Password policies enforced
  • Risk-based authentication configured
  • Session timeouts verified
  • MFA enrollment working

Common Issues

Issue Solution
MFA not prompting Check policy assignment and factor status
Session timeout issues Verify browser settings and cache
Password policy conflicts Review policy priority order

Security Best Practices

  • Regular policy review and updates
  • Monitor authentication patterns
  • Document policy exceptions
  • Maintain incident response plan
  • Regular security assessments
