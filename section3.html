User and Group Management

⏱️ 20-30 minutes

Section Prerequisites

  • Completed AD integration
  • Okta Administrator access
  • Planned user structure
  • Group naming convention

Part 1: Creating Okta-Managed Users

  1. Access User Management

    Navigate to Directory > People in the Okta admin console

    Use the bulk import template if you need to create multiple users at once.
  2. Create First Test User

    Click "Add Person" and enter details for Test User 1:

    First Name: Test1 Last Name: User1 Username: test1@yourdomain.com Primary email: test1@yourdomain.com Password: Set by admin
    Ensure "User must change password on first login" is checked.
  3. Create Second Test User

    Repeat the process for Test User 2:

    First Name: Test2 Last Name: User2 Username: test2@yourdomain.com Primary email: test2@yourdomain.com Password: Set by admin

Part 2: Creating Okta-Managed Groups

  1. Access Group Management

    Navigate to Directory > Groups

  2. Create IT-Admin Group

    Click "Add Group" and configure:

    • Name: IT-Admin
    • Description: IT Administrators group for elevated access
    • Group Type: Okta Managed
  3. Create General-Users Group

    Add another group:

    • Name: General-Users
    • Description: Standard users with basic access
    • Group Type: Okta Managed
  4. Assign Users to Groups
    1. Add Test1 User to IT-Admin group:
      • Select IT-Admin group
      • Click "Manage People"
      • Search and add Test1 User
    2. Add Test2 User to General-Users group:
      • Select General-Users group
      • Click "Manage People"
      • Search and add Test2 User

Part 3: Group Rules and Automation

  1. Create Group Rules

    Set up automated group assignments:

    1. Navigate to Directory > Groups > Rules
    2. Click "Add Rule"
    3. Configure rule conditions:
      IF User Profile Property "Department" EQUALS "Information Technology" THEN Add to Group "IT-Admin"

Verification Checklist

  • Test1 User created successfully
  • Test2 User created successfully
  • IT-Admin group created
  • General-Users group created
  • Users assigned to correct groups
  • Group rules functioning as expected
  • Users can log in successfully

Common Issues

Issue Solution
User creation fails Check for duplicate usernames or email addresses
Group assignment issues Verify group rule conditions and user attributes
User cannot log in Check user status and password requirements

Best Practices

  • Use consistent naming conventions for groups
  • Document group purposes and access levels
  • Regularly audit group memberships
  • Use group rules for automated management
  • Implement principle of least privilege
