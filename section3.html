User and Group Management

⏱️ 20-30 minutes

Section Prerequisites

  • Completed AD integration
  • Okta Administrator access
  • Planned user structure
  • Group naming convention

Part 1: Configure AD Import Settings

  1. Access Import Settings
    1. Navigate to Directory > Directory Integrations
    2. Click Active Directory
    3. Select the Provisioning tab
    4. Click Integration in the Settings list
  2. Configure User Import Settings
    1. In the Import Settings area:
      • Verify "User OUs connected to Okta" settings
      • Keep default User Filter: sAMAccountType=805306368
      • Verify Group OUs connected to Okta
      • Keep default Group Filter: objectCategory=group
    Changing default filter queries can result in deprovisioning users. Only modify filters if absolutely necessary.
  3. Configure Import Schedule
    1. Click "To Okta" in the Settings list
    2. Click Edit in the General area
    3. Set Schedule Import to daily
    4. Keep Username format as User Principal Name (UPN)
    5. Select "Create and update users on login" for JIT Provisioning
    Import notifications will be sent to administrators when changes are detected during scheduled imports.
  4. Configure Matching Rules
    1. In User Creation & Matching section, click Edit
    2. Select matching criteria:
      • Match by Okta username format
      • Enable "Auto-confirm exact matches"
    Matching rules prevent duplicate user accounts when importing from AD.
  5. Start Initial Import
    1. Return to Directory Integrations
    2. Click "Import Now" to start the process
    3. Monitor the import progress
    4. Review imported users in Directory > People

Import Verification Checklist

  • Correct OUs selected for import
  • Import filters properly configured
  • Schedule settings confirmed
  • Matching rules verified
  • Initial import completed successfully
  • Users appearing in Okta directory
  • Group memberships correct

Common Import Issues

Issue Solution
Missing users Check OU selection and user filters
Duplicate users Review matching rules configuration
Incorrect attributes Verify AD attribute mappings
Import errors Check import logs for specific error messages

Part 2: Creating Okta-Managed Users

  1. Access User Management

    Navigate to Directory > People in the Okta admin console

    Use the bulk import template if you need to create multiple users at once.
  2. Create First Test User

    Click "Add Person" and enter details for Test User 1:

    First Name: Test1 Last Name: User1 Username: test1@yourdomain.com Primary email: test1@yourdomain.com Password: Set by admin
    Ensure "User must change password on first login" is checked.
  3. Create Second Test User

    Repeat the process for Test User 2:

    First Name: Test2 Last Name: User2 Username: test2@yourdomain.com Primary email: test2@yourdomain.com Password: Set by admin

Part 3: Creating Okta-Managed Groups

  1. Access Group Management

    Navigate to Directory > Groups

  2. Create IT-Admin Group

    Click "Add Group" and configure:

    • Name: IT-Admin
    • Description: IT Administrators group for elevated access
    • Group Type: Okta Managed
  3. Create General-Users Group

    Add another group:

    • Name: General-Users
    • Description: Standard users with basic access
    • Group Type: Okta Managed
  4. Assign Users to Groups
    1. Add Test1 User to IT-Admin group:
      • Select IT-Admin group
      • Click "Manage People"
      • Search and add Test1 User
    2. Add Test2 User to General-Users group:
      • Select General-Users group
      • Click "Manage People"
      • Search and add Test2 User

Part 4: Group Rules and Automation

  1. Create Group Rules

    Set up automated group assignments:

    1. Navigate to Directory > Groups > Rules
    2. Click "Add Rule"
    3. Configure rule conditions:
      IF User Profile Property "Department" EQUALS "Information Technology" THEN Add to Group "IT-Admin"

Verification Checklist

  • Test1 User created successfully
  • Test2 User created successfully
  • IT-Admin group created
  • General-Users group created
  • Users assigned to correct groups
  • Group rules functioning as expected
  • Users can log in successfully

Common Issues

Issue Solution
User creation fails Check for duplicate usernames or email addresses
Group assignment issues Verify group rule conditions and user attributes
User cannot log in Check user status and password requirements

Best Practices

  • Use consistent naming conventions for groups
  • Document group purposes and access levels
  • Regularly audit group memberships
  • Use group rules for automated management
  • Implement principle of least privilege
