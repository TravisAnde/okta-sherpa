Active Directory Integration

⏱️ 30-45 minutes

Section Prerequisites

  • Domain Administrator access to DC01
  • Okta Administrator access
  • Outbound connectivity from DC01 to Okta
  • Service Account for AD integration
Create a dedicated service account in Active Directory before proceeding. This account will need:
  • Password never expires
  • Cannot change password
  • User cannot be locked out
  1. Prepare Active Directory Service Account

    On DC01, create a new service account:

    New-ADUser -Name "svc-okta-sync" -AccountPassword (Read-Host -AsSecureString "Input Password") -Enabled $true -PasswordNeverExpires $true -CannotChangePassword $true
    Required AD Permissions for service account:
    • Read all user attributes
    • Read all group attributes
    • Read all computer attributes
    • Reset passwords (if password sync is required)
  2. Download Okta AD Agent

    In your Okta admin console:

    1. Navigate to Directory > Directory Integrations
    2. Click "Add Active Directory"
    3. Download the Okta AD agent installer
    4. Transfer the installer to DC01
  3. Install AD Agent on DC01

    On DC01:

    1. Run the installer with Administrator privileges
    2. Select default installation path
    3. When prompted, provide:
      • Your Okta org URL
      • Admin credentials
      • Service account credentials
    Ensure DC01 can reach these URLs: *.okta.com *.oktacdn.com *.cloudflare.com
  4. Configure AD Agent Settings

    In the Okta admin console:

    1. Set import settings:
      • Select OUs to import
      • Configure user matching rules
      • Set group filter rules
    2. Configure provisioning settings:
      • Enable/disable password sync
      • Set account deactivation rules
      • Configure attribute mapping
  5. Verify Initial Import

    Monitor the initial synchronization:

    1. Check Directory > People for imported users
    2. Verify Directory > Groups for imported groups
    3. Review import errors in System Log

Verification Checklist

  • Service account created with correct permissions
  • AD agent installed successfully on DC01
  • Agent showing as "Active" in Okta console
  • Users imported successfully
  • Groups imported successfully
  • Attribute mappings verified
  • No critical errors in System Log

Common Issues

Issue Solution
Agent fails to connect Check firewall rules and proxy settings
Import errors Verify service account permissions and OU access
Password sync issues Ensure service account has password reset rights
Missing users/groups Review import filters and OU selection
Best Practice: Install a second AD agent on DC02 for redundancy after verifying the primary agent is working correctly.
