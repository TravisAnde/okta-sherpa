Part 1: LinkedIn Integration

  1. Create LinkedIn Account
    1. Open Microsoft Edge on CL03
    2. Navigate to https://www.linkedin.com/signup
    3. Create a new account using a demo email
    4. Complete phone & email verification
    Do not use your personal LinkedIn account. Create a new demo account specifically for this integration.
  2. Add LinkedIn Integration in Okta
    1. Navigate to Applications > Applications
    2. Click "Browse App Catalog"
    3. Search for "LinkedIn"
    4. Select "LinkedIn (SWA)"
    5. Click "Add Integration"
  3. Configure LinkedIn SSO
    1. On the Applications > Applications Section of the Okta Admin Console:
      • Click on "Browse App Catalog"
      • Search for "LinkedIn"
      • Select LinkedIn SWA
      • Click "Add Integration"
    2. LinkedIn Application Settings:
      • Deselect "Automatically log in when user lands on login page"
      • Click "Next"
      • Select "Users share a single username and password set by administrator"
    3. Enter your demo LinkedIn credentials
    4. Ensure "Password reveal" is unchecked
    5. Click Done
    Shared credentials allow multiple users to access LinkedIn without knowing the actual login details.
  4. Test LinkedIn Integration
    1. Assign the application:
      • Navigate to Directory > People
      • Search for your user "Adam Wally"
      • Click "Assign Applications"
      • Select LinkedIn
      • Click "Save and go back"
    2. Test the integration:
      • Switch to CL03 and Navagate to your Okta Dashboard for Adam Wally
      • Click the LinkedIn tile
      • Verify successful LinkedIn login
    Always sign out of LinkedIn after testing to avoid session conflicts when testing with different users.

LinkedIn Integration Checklist

  • LinkedIn demo account created
  • SWA integration configured
  • Shared credentials set up
  • Application assigned to test user
  • SSO login successful

Part 2: Bookmark Applications

Bookmark apps will be used later to demonstrate Device Assurance and Phishing Resistant authentication policies.
  1. Create Weather Bookmark App
    1. Navigate to Applications > Applications
    2. Click "Browse App Catalog"
    3. Search for "Bookmark"
    4. Click "Bookmark App"
    5. Click "Add Integration"
    6. Configure the following:
      • Application label: Forecast
      • URL: https://weather.com/
      • Uncheck "Request Integration"
      • Uncheck "Application Visibility"
    7. Click Done
  2. Create Flights Bookmark App
    1. Navigate to Applications > Applications
    2. Click "Browse App Catalog"
    3. Search for "Bookmark"
    4. Click "Bookmark App"
    5. Click "Add Integration"
    6. Configure the following:
      • Application label: Flights
      • URL: https://webtrak.emsbk.com/syd3
      • Uncheck "Request Integration"
      • Uncheck "Application Visibility"
    7. Click Done

Bookmark Apps Checklist

  • Weather bookmark app created
  • Flights bookmark app created
  • URLs configured correctly
  • Application labels set
  • Integration settings verified

Common Issues

Issue Solution
LinkedIn session conflicts Sign out of LinkedIn completely between tests
Bookmark apps not appearing Verify application visibility settings
SSO not working Check shared credential configuration
End User Dashboard access Verify user has application assignments

Implementation Best Practices

  • Use dedicated demo accounts for testing
  • Document all application credentials
  • Test integrations with multiple users
  • Keep bookmark URLs up to date
  • Regularly verify SSO functionality
