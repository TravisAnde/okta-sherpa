Application Integration

⏱️ 45-60 minutes

Section Prerequisites

  • Administrator access to target applications
  • Application credentials/licenses
  • SSL certificates (if required)
  • Application URLs

Applications to be Integrated

Application Integration Type Features Estimated Time
Microsoft 365 WS-Fed + SCIM SSO + Provisioning 15 min
Salesforce SAML + SCIM SSO + Provisioning 10 min
Slack SAML + SCIM SSO + Provisioning 10 min
Zoom SAML SSO Only 5 min
AWS SSO SAML SSO + Role Assignment 15 min

Microsoft 365 Integration

  1. Add Microsoft 365 Integration
    1. Navigate to Applications > Applications
    2. Click "Browse App Catalog"
    3. Search for "Microsoft Office 365"
    4. Click "Add Integration"
  2. Configure SSO Settings
    1. Enter Office 365 domain
    2. Configure WS-Federation settings
    3. Set up certificate
    4. Configure attribute mappings
    Save the PowerShell commands provided for Azure AD configuration.
  3. Configure Provisioning
    1. Enable provisioning
    2. Enter Azure AD admin credentials
    3. Configure user attribute mappings
    4. Set up group push if needed

Salesforce Integration

  1. Add Salesforce Integration
    1. Browse App Catalog
    2. Select Salesforce
    3. Choose integration type
  2. Configure SAML Settings
    1. Download Okta certificate
    2. Configure Salesforce SSO settings
    3. Set up attribute mappings

Remaining Applications

  1. Configure Slack Integration

    Follow Slack-specific configuration guide in the App Catalog

  2. Set up Zoom

    Configure basic SAML integration for Zoom

  3. AWS SSO Setup

    Configure AWS SSO with role mapping

Verification Checklist

  • Microsoft 365 SSO working
  • Salesforce integration complete
  • Slack provisioning configured
  • Zoom SSO tested
  • AWS roles assigned
  • User access verified for each app
  • Group assignments tested

Common Issues

Issue Solution
SAML configuration errors Verify certificate and endpoint settings
Provisioning failures Check API credentials and permissions
Attribute mapping issues Compare source and target attributes

Best Practices

  • Test with limited user group first
  • Document all configuration settings
  • Setup monitoring for SSO health
  • Configure app sign-on policies
  • Regular testing of SSO flows
